How fixup commits work

--fixup tags a commit to be folded into an earlier one; --autosquash does the folding during rebase - no messages to write, history stays clean.

# attach a fix to an earlier commit (makes a "fixup!" commit)
git commit --fixup abc1234

# reorder and fold every fixup! into its target
git rebase -i --autosquash <base>

# make --autosquash the default
git config --global rebase.autoSquash true

You never write a message yourself. GIT copies the target commit's subject verbatim and prefixes it with fixup! , so a fix attached to a commit titled Add login form looks like this:

# the commit GIT creates for you
fixup! Add login form

That fixup! prefix is exactly what --autosquash matches on to find the target during rebase.

💡 Never merge a stray fixup! commit - if it reaches the target branch, the squash never happened.
