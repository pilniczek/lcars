💡 Makes GIT open VSCode for commit messages, interactive rebases, and conflict markers instead of dropping you into vim or nano.

The problem

When GIT needs you to edit text - a commit message without -m, an interactive rebase todo list, a merge conflict prompt - it launches its default editor. On most machines that is vim or nano, and if you don't live in vim you get stuck in a modal editor with no obvious way out.

What the settings do

[core]
    editor = code --wait

Now GIT opens the file in VSCode. The --wait flag is the critical part: it tells code to block until you close the tab, so GIT waits for you to finish editing instead of seeing an instantly-returned empty file and aborting.

If code isn't on your PATH (common on Windows), point at the full path instead:

[core]
    editor = "C:\\Users\\__PLACEHOLDER_USER__\\AppData\\Local\\Programs\\Microsoft VS Code\\bin\\code" --wait
