Husky lets you define GIT hooks that live in the repository itself and are shared across the whole team. As covered in the GIT hooks article, native hooks live in .git/, so they are local to a clone and are not committed or distributed for security reasons - every developer would otherwise have to wire up .git/hooks by hand. Husky closes that gap.

You declare the hooks you want once, and Husky installs them into each developer's local repository automatically on npm i (via the prepare script). The most common use is running lint or format checks on a pre-commit hook so problems are caught before they ever land in history. For setup and configuration, see the official docs: https://typicode.github.io/husky/.
