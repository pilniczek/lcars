💡 One command to dump every change into a throwaway WIP commit. Switch branches, pull, or step away without git stash juggling - and it skips hooks, so it's instant.

The problem

You're mid-change and need to switch branches, pull, or just save a checkpoint. git stash works, but the stash list is easy to forget and awkward to read. A plain commit means inventing a message and waiting on pre-commit hooks you don't care about yet.

The alias

[alias]
    # commit all changes to a WIP commit
    wip = !git add --all && git commit -m WIP --no-verify && git pf

git wip stages everything (tracked and untracked) and commits it as WIP. --no-verify skips Husky and other pre-commit / commit-msg hooks, so the checkpoint is instant. The trailing git pf (push --force-with-lease) also pushes the checkpoint to your tracking branch as an off-machine backup - safe because it only ever overwrites your own branch and never merges anywhere.

Undoing it

git reset HEAD~

Guard the pipeline

A WIP commit is a private checkpoint, never something to merge. Catch stray ones in CI by failing the pipeline when a WIP commit reaches the branch.

How to set
